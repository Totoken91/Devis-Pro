1 Ce que fait le site, concrètement

Deviso est un outil web qui permet aux freelances de créer, envoyer et suivre leurs devis en quelques minutes — sans Word, sans Excel, sans PDF envoyé par email.

❌ Sans Deviso

  1. Ouvrir Word, recopier l'entête à la main
  2. Taper les lignes de prestation une par une
  3. Calculer la TVA sur une calculette
  4. Exporter en PDF, l'envoyer par email
  5. Attendre… sans savoir si le client l'a ouvert
  6. Relancer à la main 1 semaine plus tard
  7. Chercher dans sa boite mail "où est le devis signé"

✅ Avec Deviso

  1. Se connecter, sélectionner le client
  2. Ajouter les lignes de prestation (ou piocher dans le catalogue sauvegardé)
  3. TVA calculée automatiquement
  4. Cliquer "Envoyer" → le client reçoit un lien
  5. Notif dès que le client ouvre la page
  6. Relance automatique à J+3 et J+7 si pas de réponse
  7. Le client signe directement depuis son navigateur

Ce que le site NE fait PAS

Deviso n'est pas un logiciel de comptabilité, pas un CRM, pas un outil de gestion de projet. Il fait une seule chose : le devis, de la création à la signature. C'est sa force.

4 Roadmap — 3 Phases

Pas de phase 4, pas de mobile, pas d'équipe. 3 phases, un seul fondateur, un produit qui tourne.

PHASE 1 — MVP (Mois 1 à 3) Objectif : 50 utilisateurs actifs
Ce qu'on apprend
Créer un compte utilisateur, protéger des routes, lier des données à un user. C'est le cœur de tout SaaS.
Auth
Inscription email + mot de passe via Supabase Auth. Vérification email automatique. Connexion OAuth Google (bouton "Continuer avec Google"). Sessions gérées côté serveur via @supabase/ssr.
Profil
Nom, prénom, nom d'entreprise, email, téléphone, adresse, SIRET (optionnel), logo uploadé. Ces infos s'affichent automatiquement sur tous les devis.
Clients
Carnet d'adresses simple : créer un client (nom, email, entreprise, adresse). Chaque client est lié au user_id — invisible des autres utilisateurs.
Créateur de devis
Sélectionner un client, ajouter des lignes (label, quantité, prix unitaire, TVA). Calcul automatique HT / TVA / TTC. Champ remise optionnel. Numérotation auto (ex: DEV-2026-001).
Templates
3 designs visuels (minimaliste, classique, moderne). Couleur de marque personnalisable. Logo inclus automatiquement.
Partage
Lien unique public /q/[token] — le client ouvre la page dans son navigateur sans compte. Page responsive, propre, professionnelle.
Export PDF
Génération PDF serveur via @react-pdf/renderer ou puppeteer. Téléchargement direct, même template que la page web.
Dashboard
Liste des devis avec statut (brouillon / envoyé / accepté / refusé / expiré), montant, client, date. Filtres de base.
Feedback
Un simple bouton "Envoyer un retour" dans l'interface. Pas de tests utilisateurs organisés — les retours arrivent naturellement.
PHASE 2 — Monétisation (Mois 4 à 6) Objectif : 30 abonnés payants
Ce qu'on apprend
Intégrer Stripe, gérer les webhooks, restreindre des features selon le plan. La vraie mécanique d'un SaaS.
Stripe
Stripe Checkout pour le paiement. Stripe Customer Portal pour que l'utilisateur gère lui-même son abonnement (annulation, changement de CB). Webhooks pour synchroniser le statut en base (customer.subscription.created, invoice.paid, customer.subscription.deleted).
Limitation plan
Vérification du plan à chaque création de devis. Si gratuit et déjà 3 devis ce mois-ci → modale d'upgrade claire et non-agressive. Compteur visible dans le dashboard.
Suivi ouverture
Pixel de tracking 1×1 sur la page client. À l'ouverture → event enregistré en base (quote_events), statut du devis passe à "vu", notification email envoyée au freelance via Resend.
Relances auto
Job planifié (Vercel Cron Jobs) : chaque nuit, cherche les devis envoyés sans réponse. Envoie un rappel au client à J+3 et J+7. Désactivable par le freelance. Log de toutes les relances.
Signature
Bouton "Accepter ce devis" sur la page client. Collecte nom + date + IP. Stocké en base. Statut passe à "accepté". Email de confirmation envoyé aux deux parties.
Emails
Via Resend + templates react-email : confirmation inscription, devis envoyé au client, devis ouvert par le client, devis signé, relance, confirmation paiement Stripe.
Page pricing
Landing page dédiée avec les deux plans, FAQ, toggle mensuel / annuel (−20%). Call to action vers Stripe Checkout.
PHASE 3 — Rétention (Mois 7 à 12) Objectif : 100–200 abonnés · MRR 1 500–3 000 €
Ce qu'on apprend
Réduire le churn, augmenter la valeur perçue, faire revenir les utilisateurs. Le vrai défi d'un SaaS mature.
Catalogue services
Sauvegarder ses prestations fréquentes (label, prix, TVA). Les réutiliser en 1 clic lors de la création d'un devis. Gain de temps immédiat, rétention naturelle.
CRM léger
Fiche client enrichie : historique des devis, montant total facturé, taux d'acceptation, dernière interaction. Rien de plus.
Stats dashboard
Taux d'acceptation global, montant moyen d'un devis, temps moyen avant réponse, revenus estimés du mois. Graphique simple.
Referral
Code de parrainage unique par utilisateur. Si un ami s'inscrit via le lien et souscrit : 1 mois offert à chacun. Géré via Stripe coupons.
Mentions légales
Bloc CGV optionnel à ajouter en bas de chaque devis. Template type micro-entrepreneur pré-rempli, personnalisable.
Fin de roadmap
Pas de phase 4. Pas d'équipe. Pas d'API publique. Pas de mobile natif. Si le produit fonctionne, les prochaines directions seront dictées par les vrais utilisateurs.

5.2 Chaque brique en détail

Frontend + Backend
⚡ Next.js 14 (App Router)
Framework React full-stack. Un seul repo pour le front et l'API. Server Components pour les pages, Route Handlers pour les webhooks Stripe, Server Actions pour les mutations de données. Deploy natif sur Vercel.
✓ Gratuit · npm create next-app
Authentification + BDD + Fichiers
🟢 Supabase
Fournit tout-en-un : Auth (email/password + OAuth Google via @supabase/ssr), PostgreSQL managé avec Row Level Security (RLS) pour isoler les données par utilisateur, Storage pour les logos uploadés.
✓ Gratuit jusqu'à 500 MB · Pro à 25$/mois si besoin
Paiements + Abonnements
💳 Stripe
Stripe Checkout pour la page de paiement hébergée. Stripe Customer Portal pour la gestion self-service. Webhooks pour synchroniser les événements en base (subscription créée, payée, annulée). stripe npm package côté serveur uniquement.
✓ 0 € fixe · 1,5% + 0,25€ par transaction (EU)
Emails transactionnels
📧 Resend + React Email
API email simple avec SDK TypeScript. Templates écrits en React via react-email — mêmes composants que l'app. Emails envoyés : inscription, devis client, notification ouverture, relances, confirmation paiement.
✓ 100 emails/jour gratuit · 20$/mois pour 50k emails
Génération PDF
📄 @react-pdf/renderer
Génère des PDF côté serveur via une Route Handler Next.js. Même structure que les templates web. Renvoie un application/pdf directement téléchargeable. Alternative : Puppeteer (plus lourd, meilleur rendu).
✓ Open source · Gratuit
Jobs planifiés
⏰ Vercel Cron Jobs
Tâches planifiées via vercel.json. Utilisé pour : envoyer les relances automatiques chaque nuit (J+3, J+7), expirer les devis sans réponse au bout de 30 jours. Déclenche une Route Handler sécurisée par un secret header.
✓ Inclus dans Vercel Pro · 20$/mois
Styling
🎨 Tailwind CSS + shadcn/ui
Tailwind pour le style utilitaire. shadcn/ui pour les composants UI (modales, boutons, formulaires, toasts) — copiés directement dans le projet, pas de dépendance externe fragile.
✓ Gratuit · Open source
Deploy + Hébergement
▲ Vercel
Deploy automatique à chaque push Git. Preview URLs pour chaque PR. Edge Network CDN mondial. Analytics inclus. Variables d'environnement sécurisées. Cron Jobs (Pro). Logs en temps réel.
✓ Gratuit (Hobby) → Pro 20$/mois dès Cron nécessaire
Monitoring erreurs
🐛 Sentry
Capture automatique des erreurs front et back. Stack traces, contexte utilisateur, alertes email. Intégration Next.js native via @sentry/nextjs. Indispensable dès qu'il y a des utilisateurs réels.
✓ Gratuit jusqu'à 5k erreurs/mois
Typage + Qualité code
🔷 TypeScript + Zod
TypeScript partout pour éviter les bugs à l'exécution. Zod pour valider les données entrantes (formulaires, webhooks Stripe, params d'URL). Types Supabase auto-générés depuis le schéma DB.
✓ Gratuit · Inclus dans Next.js
Gestion de l'état
🔄 React Query (TanStack)
Fetching, caching et invalidation des données côté client. Utilisé pour les listes de devis, clients, prestations. Évite la gestion manuelle des états de chargement / erreur.
✓ Gratuit · Open source
Variables d'environnement
🔐 .env.local
Toutes les clés sensibles dans des variables d'environnement, jamais dans le code. Clés nécessaires : SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, RESEND_API_KEY, CRON_SECRET.
✓ Géré par Vercel en production

S6

S7
Stripe : intégration Checkout + Customer Portal + webhooks. Table subscriptions. Limitation à 3 devis/mois pour le plan gratuit. Page pricing.
S8
Tracking + Signature : pixel de tracking ouverture. Notification email au freelance. Bouton signature côté client. Confirmation aux deux parties.
S9
Relances auto : Vercel Cron Job nocturne. Logique J+3 / J+7. Table reminders. Emails de relance via Resend. Toggle d'activation par devis.
S10
Polish + Sentry : intégration Sentry, gestion des edge cases (devis expiré, webhook Stripe échoué, email bounced), états de chargement, messages d'erreur clairs.
S11
Landing page : page d'accueil publique avec démo visuelle, pricing, FAQ, CTA. SEO de base (meta tags, sitemap). Bouton feedback dans l'app.

Lancement : ProductHunt, Reddit r/freelance_fr, groupes Facebook freelances. Objectif : 50 inscrits. Observer les comportements réels et itérer.
